Módulo 1: ¿Cómo se organiza un proyecto PHP real?

📌 ¿Qué vamos a aprender?

Hasta ahora, probablemente has escrito tus ejercicios en un solo archivo: ejercicio1.php. Esto funciona para tareas pequeñas, pero imagina un proyecto con 50 archivos y 5 compañeros de equipo... ¡sería un caos!

Aquí aprenderás a organizar tu código como lo hacen los profesionales, usando patrones probados en la industria.

1.1 El problema del "todo en un archivo"

❌ El enfoque "Espagueti" (Lo que probablemente haces ahora)

<?php
// Todo mezclado: conexión, lógica, validación y HTML
$conexion = new mysqli("localhost", "root", "", "mi_bd");

if ($_POST) {
    $nombre = $_POST['nombre'];
    // Guardar en base de datos directamente
    $conexion->query("INSERT INTO usuarios (nombre) VALUES ('$nombre')");
    echo "Usuario registrado";
}
?>
<form method="POST">
    <input name="nombre">
    <button>Registrar</button>
</form>

Problemas: 🔴 Si cambias el diseño, tocas la lógica. 🔴 Copias y pegas la conexión en cada archivo. 🔴 Es vulnerable a ataques.

1.2 La solución: Separar responsabilidades (MVC)

✅ La analogía de la Oficina

Piensa en tu proyecto como una oficina. No todos hacen todo:

  • 👔 El Recepcionista (Controlador): Recibe la petición del usuario, verifica permisos y dirige el tráfico.
  • 📊 El Contador (Modelo): Habla con la base de datos. Sabe cómo guardar y buscar datos, pero no sabe nada de HTML.
  • 🎨 El Diseñador (Vista): Solo se encarga de mostrar la información en HTML. No toma decisiones ni consulta la base de datos.

📊 Diagrama: Flujo de una petición MVC

1.3 La estructura de carpetas profesional

✅ Organización por carpetas

mi_proyecto/
├── public/              ← 🌐 ÚNICA carpeta visible desde el navegador
│   ├── index.php        ← El "Recepcionista" (Front Controller)
│   └── assets/          ← CSS, JS, Imágenes
├── app/                 ← 🧠 El "cerebro" (inaccesible desde la web)
│   ├── Controllers/     ← Lógica de flujo
│   ├── Models/          ← Lógica de datos
│   └── Views/           ← Archivos HTML
├── config/              ← ⚙️ Contraseñas y configuración
└── storage/             ← 📦 Logs de errores y archivos subidos

1.4 Autoloading: No más "require" por todos lados

✅ La función mágica de PHP

En lugar de escribir 20 require_once, usamos spl_autoload_register. Le dice a PHP: "Cuando veas new App\Models\Usuario, busca automáticamente en app/Models/Usuario.php".

// config/autoload.php
spl_autoload_register(function ($nombreClase) {
    $archivo = __DIR__ . '/../app/' . str_replace('\\', '/', $nombreClase) . '.php';
    if (file_exists($archivo)) {
        require_once $archivo;
    }
});
