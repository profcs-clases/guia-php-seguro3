✅ Checklist de Seguridad Imprimible

📚 Glosario Simplificado

  • Front Controller: Un único archivo (index.php) que recibe todas las peticiones y decide qué hacer. Es como la recepción de un edificio.
  • SQL Injection (SQLi): Ataque donde el usuario introduce código SQL en un formulario para manipular la base de datos. Se previene con Consultas Preparadas.
  • XSS (Cross-Site Scripting): Ataque donde se inyecta código JavaScript malicioso que se ejecuta en el navegador de otros usuarios. Se previene con htmlspecialchars.
  • CSRF: Engañar a un usuario logueado para que realice una acción sin su consentimiento. Se previene con Tokens CSRF.
  • Hashing: Proceso de una sola vía para convertir una contraseña en una cadena de texto ilegible. No se puede "desencriptar", solo se puede comparar.
  • Middleware: Un "filtro" que se ejecuta antes de llegar a la lógica principal, ideal para verificar si un usuario tiene permiso (rol) para ver una página.

🚀 Próximos Pasos (Cultura Profesional)

Estos temas no son obligatorios para aprobar este curso, pero dominarlos te dará una ventaja enorme en tu primer empleo:

  • Composer: El gestor de paquetes de PHP. Aprende a usarlo para instalar librerías en lugar de copiar y pegar código.
  • Archivos .env: Nunca escribas contraseñas en el código. Usa un archivo .env (que no se sube a Git) para guardarlas.
  • Git y Pull Requests: Trabaja en ramas separadas (feature/mi-cambio) y nunca hagas cambios directamente sobre la rama main.
  • Xdebug: Una herramienta que te permite detener el código línea por línea en tu editor (VS Code) para ver exactamente qué está fallando, sin usar var_dump.
